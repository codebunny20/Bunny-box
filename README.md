# Bunny box

WELCOME TO THE BUNNY BOX REPOSITORY!
This repository contains the source code for the Bunny Box app, which is a tool box for all of your tool needs. The app is designed to be simple and easy to use, with a focus on providing useful functionality to the user. The app is intended to be a tool box for all of the user's tool needs(sadly this will only work on Windows, so my apologies to Mac and Linux users), with a wide variety of tools available for different purposes. i use these tools daily and they have been very useful to me, so i wanted to share them with others who may find them useful as well. The app is still in development, so there may be some bugs and missing features, but i am constantly working on improving it and adding new tools. If you have any suggestions for new tools or features, please feel free to open an issue or submit a pull request. Thank you for checking out the Bunny Box repository!

## current tools[8] [6/16/26]
accessibility tools:
- TTS tool: a tool that opens a popup window where the user can enter text and have it read aloud using the built-in text-to-speech functionality of Windows.

misc system tools:
- disk info: a tool that shows the user information about their disk drives, such as the total size, free space, and used space.
- pc uptime: a tool that shows the user how long their computer has been running since the last boot.
- process counter: a tool that shows the user the number of processes currently running on their computer.

reminder tools:
- reminder: a tool that allows the user to set a reminder for a specific date and time

system spec tools:
- simple system specs: a tool that shows minimum system specifications for yourer computer, such as the CPU, RAM, GPU, and OS.
- system spec details: a tool that dumps all of the system information to a text file, including the CPU, RAM, GPU, OS, and much more detailed information about each component.

- view system specs: a tool that opens the specs.txt file that is generated by the system spec details tool, allowing the user to quickly view the detailed system information. in a txt file in note pad or, once its been generated once you can open it with the view system specs tool, which will open the file directly in the app itself, allowing you to view the detailed system information without having to open it in a separate program.


# workflow
the apps workflow is as follows:
1. The user opens the app and is presented with a dashboard.
   - The dashboard displays different categories of tools, such as "System spec Tool", "accessibility Tools", "File Management Tools", etc.
   - Each category contains a list of tools that the user can select from.
   - 
2. The user can navigate to a tool category and select a tool to use.
   - there will be more tools added in the future, but for now there are only a few tools available. the user can select a tool by clicking on it, which will open a new window with the tool's interface. the user can then interact with the tool and use its functionality as needed.

3. The user can press the corresponding button to run the tool.

4. The app will execute the tool and display the results to the user in various formats, depending on the tool's functionality.
